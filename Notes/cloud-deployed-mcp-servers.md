# Cloud Deployed MCP Servers

## Goal

We wish to support cloud deployment of MCP servers via:

```wl
CloudDeploy[MCPServerObject[...], ...]
```

## Implementation Details

We should define the ``CloudDeploy`` behavior via ``UpValues`` on ``MCPServerObject``, e.g.

```wl
MCPServerObject /: CloudDeploy[ obj_MCPServerObject, args___ ] := catchTop[
    cloudDeployMCPServer[ obj, args ],
    MCPServerObject
];
```

The output of the ``CloudDeploy`` should be a ``CloudObject`` that corresponds to a directory. We should also deploy the following into that directory:

* ``/mcp`` - the actual deployed MCP server endpoint

* ``/index.html`` - a landing page that provides information about the server

* ``/admin/index.html`` - an admin page that allows the owner to create/delete API keys, monitor usage, enable/disable server, etc.

* ``/api/admin`` - an API deployed for processing UI actions on the admin page

* ``/files/`` - a location for artifacts generated by the server, e.g. cloud deployed notebooks for MCP apps, images, etc.

* ``/logs/`` - a place to store log files for the server (future; not needed for initial implementation)

Any other APIs needed to support ``/index.html`` can go into ``/api`` as needed.

Other files such as scripts, CSS, and images needed for implementation should be placed in a sensible location (up to the developer).

We should also expose a ``CloudDeployMCPServer`` function in the AgentTools context that allows one to deploy *just* the MCP server endpoint with whatever path, permissions, etc they want.

### Landing Page

The landing page should include the following:

* Basic information about the server (name, available tools, etc)

* Click-to-copy fields to easily configure clients

* Basic instructions for usage

* Link to admin page

### Permissions

We should respect whatever ``Permissions`` setting is passed via ``CloudDeploy`` when deploying ``index.html``. However, the other deployments have special requirements that override the user's ``Permissions`` specification:

* ``/admin/index.html`` - should always be private, since it's only intended for the owner

* ``/api/admin`` - should also be private, using credentials from the admin page

* ``/mcp`` - starts off with same permissions as ``index.html`` but the admin page can create/remove permissions keys

### Paclet Layout

Since this will share a lot of functionality with ``StartMCPServer``, we should move this functionality into a ``Server/`` directory.

* ``Kernel/Server/Server.wl`` - entry point that loads other files

* ``Kernel/Server/Shared.wl`` - functionality shared by local and remote server implementations, e.g. ``handleMethod``

* ``Kernel/Server/Local.wl`` - ``StartMCPServer`` implementation and any other local-specific functionality

* ``Kernel/Server/Cloud.wl`` - cloud/remote functionality, including ``CloudDeployMCPServer``

### Deployment Hints

Static resources like HTML pages can be kept as paclet assets and deployed with ``CopyFile`` :

```wl
CopyFile["path/to/local.html", CloudObject["path/to/target.html", Permissions -> permissions], OverwriteTarget -> True]
```

## Basic Remote MCP Server Example

### Definitions

**Note:** These definitions were mostly copied/modified from a much older version of the AgentTools paclet and are likely out of date.

```wl
$protocolVersion = "2025-03-26";

$initResult = <|
    "protocolVersion" -> $protocolVersion,
    "capabilities" -> <|
        "logging"    -> <| |>,
        "prompts"    -> <| |>,
        "resources"  -> <| |>,
        "tools"      -> <| "listChanged" -> True |>
    |>,
    "serverInfo" -> <| "name" -> "Remote MCP Test", "version" -> "1.0.0" |>
|>;

$llmTools = <|
    "PrimeFinder" -> LLMTool[ "PrimeFinder", { "n" -> "Integer" }, Prime[ #n ] & ]
|>;

$toolList = {
    <|
        "name"        -> "PrimeFinder",
        "description" -> "Finds the nth prime",
        "inputSchema" -> <|
            "type"       -> "object",
            "properties" -> <| "n" -> <| "type" -> "integer", "description" -> "integer number" |> |>,
            "required"   -> { "n" }
        |>
    |>
};

$promptList = { };

handleMethod // ClearAll;

handleMethod[ "initialize"    , msg_, req_ ] := <| req, "result" -> $initResult |>;
handleMethod[ "ping"          , msg_, req_ ] := <| req, "result" -> { } |>;
handleMethod[ "resources/list", msg_, req_ ] := <| req, "result" -> <| "resources" -> { } |> |>;
handleMethod[ "prompts/list"  , msg_, req_ ] := <| req, "result" -> <| "prompts" -> $promptList |> |>;
handleMethod[ "prompts/get"   , msg_, req_ ] := <| req, "result" -> getPrompt[ msg, req ] |>;
handleMethod[ "tools/list"    , msg_, req_ ] := <| req, "result" -> <| "tools" -> $toolList |> |>;
handleMethod[ "tools/call"    , msg_, req_ ] := <| req, "result" -> evaluateTool[ msg, req ] |>;

(* Ignored *)
handleMethod[ method_String, _, req_ ] /; StringStartsQ[ method, "notifications/" ] := Null;
handleMethod[ _, _, KeyValuePattern[ "id" -> Null ] ] := Null;

(* Unknown method *)
e : handleMethod[ method_, msg_, req_ ] := (
    writeError[ "Unhandled method: " <> ToString[ Unevaluated @ e, InputForm ] ];
    <| req, "error" -> <| "code" -> -32601, "message" -> "Unknown method" |> |>
);


evaluateTool // ClearAll;
evaluateTool[ msg_, req_ ] := Enclose[
    Catch @ Module[ { params, toolName, args, result, string },
        params = ConfirmBy[ Lookup[ msg, "params", <| |> ], AssociationQ ];
        toolName = ConfirmBy[ Lookup[ params, "name" ], StringQ ];
        args = Lookup[ params, "arguments", <| |> ];
        result = $llmTools[ toolName ][ args ];
        If[ StringQ @ result[ "String" ], result = result[ "String" ] ];
        string = ConfirmBy[ safeString @ result, StringQ, "String" ];
        <| "content" -> { <| "type" -> "text", "text" -> string |> } |>
    ],
    throwInternalFailure
];


safeString // ClearAll;
safeString[ arg_ ] := ToString[ arg, CharacterEncoding -> "PrintableASCII" ];


responseContentType // ClearAll;
responseContentType[ ] := responseContentType @ HTTPRequestData[ ];
responseContentType[ KeyValuePattern[ "Headers" -> headers_ ] ] := responseContentType @ KeyMap[ ToLowerCase, Association @ headers ];
responseContentType[ KeyValuePattern[ "accept" -> accept_String ] ] := responseContentType @ StringTrim @ StringSplit[ accept, "," ];
responseContentType[ accept : { ___String } ] := SelectFirst[ { "application/json", "text/event-stream" }, MemberQ[ accept, # ] &, None ];
responseContentType[ ___ ] := None;


makeResponseString // ClearAll;
makeResponseString[ "application/json", result_ ] := Developer`WriteRawJSONString[ result, "Compact" -> True ];
makeResponseString[ "text/event-stream", result_ ] := "data: " <> Developer`WriteRawJSONString[ result, "Compact" -> True ] <> "

";


RunCloudMCPServer // ClearAll;
RunCloudMCPServer[ ] :=
    Catch @ Module[ { req, accept, message, method, id, result, resp },
        req = HTTPRequestData[ ];
        accept = responseContentType @ req;
        If[ ! StringQ @ accept, Throw @ HTTPResponse[ "", <| "StatusCode" -> 405 |> ] ];
        message = Developer`ReadRawJSONString @ ByteArrayToString @ req[ "BodyByteArray" ];
        If[ ! AssociationQ @ message, Throw @ HTTPResponse[ "", <| "StatusCode" -> 400 |> ] ];
        method = Lookup[ message, "method", None ];
        id = Lookup[ message, "id", Null ];
        result = handleMethod[ method, message, <| "jsonrpc" -> "2.0", "id" -> id |> ];
        resp = makeResponseString[ accept, result ];
        HTTPResponse[ StringToByteArray @ resp, <| "ContentType" -> "application/json" |> ]
    ];
```

A ``RunCloudMCPServer`` should also be exported in the AgentTools context. The actual implementation should accept an ``MCPServerObject`` as the argument and set things like ``$toolList`` appropriately.
